The Mission

作为数字媒体技术毕业生，我需要一个既能展示 3D 作品，又能证明我前端开发能力的平台。目标是：**高性能、跨设备、纯手写**。

Technical Magic

放弃了模板，选择从零构建。利用 **CSS Grid** 实现复杂的响应式布局，确保在手机和 4K 屏幕上都有完美表现。

The Challenge

As a Digital Media Technology graduate, I needed a platform that not only showcases my diverse projects but also reflects my technical proficiency in bridging the gap between design and code. The goal was to create a high-performance, responsive personal brand hub that provides a seamless user experience across all devices.

Design Strategy

Visual Identity: I chose a minimalist "Dark Mode" aesthetic for the navigation to ensure that project thumbnails stand out without visual clutter.

Information Architecture: I implemented a "Card-based" layout for project details to group related information (Video, Text, Gallery) into digestible units.

Typography: Selected clean, sans-serif typefaces to ensure maximum readability for long-form case studies.

Technical Implementation

Unlike using templates, I built this site from scratch using HTML5, CSS3, and JavaScript to ensure complete creative control.

Responsive Grid: Utilized CSS Grid and Flexbox to ensure the layout adapts from large desktop monitors to mobile screens.

Interactive Components: Developed custom dropdown menus and hover effects to enhance micro-interactions and user engagement.

Performance Optimization: Optimized assets and hosting via GitHub Pages for fast loading speeds and reliable uptime.

Key Features

Dynamic Navigation: A smooth, hover-active navigation bar with organized sub-categories for Game Dev, UI/UX, and Research.

Integrated Media Gallery:A seamless blending of video trailers and high-resolution image galleries within a single project card.

Consistent Branding: A unified color palette and spacing system applied across all sub-pages.

Bloom-In-Motion: Ticketing Web App

Role: UI/UX Designer

The Challenge

Managing event registration for university clubs can be chaotic. Traditional manual sign-ups often lead to fragmented information and high cognitive load for students. The goal was to design a mobile-friendly ticketing web app that simplifies seat selection and payment verification, ensuring a seamless experience for both organizers and attendees.

The Process

Requirement Gathering: I conducted user interviews with club representatives to map out pain points in the existing registration flow, identifying "information clarity", "registration traffic" and "seat availability" as top priorities.

Flow Optimization: Applying Human-Computer Interaction (HCI) principles, I designed an intuitive user flow that minimizes the number of steps required to complete a booking.

01. Landing Page

I curated event details and club background information to ensure students can immediately find the "Why" and "What" of the event.

02. Interactive Seating

I designed a grid-based auditorium layout, focusing on effective tap-target sizing and clear status indicators (Available vs. Occupied) to ensure effortless selection.

03. Structured Payment Flow

By utilizing Progressive Disclosure, I broke down the confirmation process into manageable segments, effectively reducing user anxiety and potential errors during checkout.

04. System Visibility

I implemented a robust "Payment Status" page that provides instant, positive feedback, ensuring the user feels confident about their successful purchase.

UI/UX Case Study

TravelBuddy: All-in-One Travel Planner

Role: UI/UX Designer

✦ The Challenge

Budget-conscious travelers struggle with fragmented tools—switching between spreadsheets for budgeting and maps for navigation. I aimed to create a seamless, integrated experience combining itinerary planning and financial tracking.

✦ User Research & Insights

Methodology: Conducted surveys to uncover real-world frustrations.

Key Finding: "Hidden costs" and "Complex syncing" were the biggest stressors. This data directly informed the app’s core budget-tracking features.

01. Flat Hierarchy

Developed wireframes with a "three-tap" rule: users can log an expense or add a destination in fewer than three taps to minimize friction.

02. Minimalist UI

Applied a clean aesthetic with high-contrast typography, ensuring readability even in bright, outdoor travel conditions.

03. Cognitive Load Reduction

Utilized intentional white space to help users focus on essential travel information without feeling overwhelmed.

04. Community Wisdom

Integrated a social layer for verified "budget tips," fostering trust and connection within the traveling community.
